## Challenges

### 1. Race Conditions — Most Persistent Technical Pattern (5/23 items)
- prep-1: TanStack Query `lgas` data undefined before render
- prep-3: Sidebar route matching prefix collision
- prep-10: TOCTOU in governance guards (count check outside transaction)
- 6-5: TOCTOU in `openDispute()` — validation outside transaction
- 6-6: Dispute state transitions vulnerable to concurrent access

Race conditions appeared in 22% of items this cycle. They span frontend (TanStack Query timing), routing (NavLink prefix matching), and backend (database TOCTOU). This is a pattern that needs a structural anti-pattern guide.

### 2. Production Infrastructure Blind Spot
Awwal's UAT found: 502 login errors, high RAM/CPU (local dev), email quota exceeded (411 queued messages), hCaptcha loading slowly. These are operational concerns that our three testing layers don't cover. Story 6-2's monitoring dashboard helps detect but doesn't prevent.

### 3. SEC-3 CORS_ORIGIN Production Crash Loop
The security hardening added `CORS_ORIGIN` to `requiredProdVars` — correct code. But deploying without setting the env var on the server caused 12+ crash-restart cycles on production (2026-03-02 18:31). Root cause: no deployment coordination between code changes that add required env vars and the production environment configuration.

### 4. "Coming Soon" Placeholder Pages
Catch-all wildcard routes (`path="*"`) per role display "coming soon" for any undefined URL. While no specific sidebar items are broken (all 11 Super Admin items are implemented), users navigating to old/bookmarked URLs see generic placeholders. Inappropriate for a government system.

### 5. View-As Partial Implementation
Story 6-7's View-As feature only renders one role correctly; other roles don't display properly. The component import mapping (`roleComponentMap`) likely has incorrect or missing lazy-loaded routes for some roles. Not blocking (direct login works) but the feature doesn't serve its debugging/demo purpose.

### 6. Active Production Bug — `text = uuid` Type Mismatch
`respondent.service.ts:325` throws `operator does not exist: text = uuid` on production. Affects the respondent registry page and likely the search functionality. Discovered via production logs during this retrospective.

### 7. Process Commitments Need Structural Enforcement
403 test requirement flagged for 4th consecutive retro. The pattern is clear: standards that depend on individual memory don't stick. Only commitments enforced by existing processes (code review catches missing tests per A8) become reflexive. Future improvements must be tooling-based, not standard-based.

## Key Insights

### 1. Layer 3 Human UAT by the Context-Holder is Irreplaceable
Automated tests catch code bugs. Adversarial review catches architectural gaps. But only a human with full project context — testing the live product end-to-end as a real user — catches the integration-level issues: email quota explosion, placeholder pages, missing data visibility, resource saturation. Awwal's single UAT session created more high-impact work than any other activity in the cycle.

### 2. Story-Shaped Work Completes; Aspirational Commitments Drift
Confirmed for the third consecutive epic. Bug fixes (prep tasks): 100%. Process improvements (standards, checklists): partial. The structural solution: frame all improvements as concrete, deliverable, story-shaped tasks with acceptance criteria. The portable playbook replaces P5 (carried 4 retros) by making it a real deliverable.

### 3. Solo Developer + AI Team Eliminates Context Fragmentation
Traditional teams distribute work across people, fragmenting context. Each person sees their slice — bugs live in the gaps between understanding. The OSLRS model — Awwal holding full context, AI agents providing specialized skills — eliminates this fragmentation. Trade-off: bus factor of 1, bandwidth ceiling. Mitigation: externalized context artifacts (retros, sprint-status, project-context, context brief).

### 4. Commit History as Retro Input
Proposed by Awwal from his other project. Story files capture the destination; commit trails capture the journey — false starts, CI failures, incident response. The SEC-3 CORS_ORIGIN crash loop is invisible in the story file but fully documented in the commit history. Implementation: selective, anomaly-driven summary tool (story-shaped prep task).

### 5. Deployment Safety Requires Env Var Coordination
Code changes that add required env vars (like SEC-3's `CORS_ORIGIN` → `requiredProdVars`) must be coordinated with the production environment. Without a pre-deploy check, the app crash-loops on startup. This is a category of issue that will recur with Epic 7 (new env vars for marketplace, CAPTCHA, etc.).

### 6. Context Continuity Across Sessions is a Force Multiplier
The wealth of institutional knowledge in a long session is lost when starting fresh. Solution: externalized context artifacts — retro documents, context brief, auto-memory, sprint-status. These transfer accumulated wisdom to new sessions without manual context loading. The context brief (`docs/team-context-brief.md`) is the bridge between sessions.
